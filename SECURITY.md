# Security Policy

This project is intended for controlled malware storage workflows. Treat all
sample files and archives as untrusted input.

## Reporting Issues

Do not attach malware samples to public issues. Report security concerns with
enough metadata to reproduce the problem safely, such as hashes, container
format, stack traces, and sanitized command output.

## Handling Expectations

- Samples should remain encrypted or compressed when possible.
- Extraction should happen only in isolated analysis environments.
- Public test fixtures must not contain live malware.
